## 一、格式化输出实战

### 1.1 调试打印的基本套路

开发中最常用的调试手段就是把变量值打出来看。以下是一个简单但完整的调试打印模板：

```c
#include <stdio.h>

int main(void)
{
    int status = -1;
    unsigned int count = 42;
    float rate = 0.85f;
    char *msg = "processing";

    // 一行打出多个关键变量（调试常用）
    printf("[DEBUG] status=%d, count=%u, rate=%.2f, msg=%s\n",
           status, count, rate, msg);
    // 输出：[DEBUG] status=-1, count=42, rate=0.85, msg=processing
}
```

### 1.2 表格化对齐输出

在实际工具中常需要以对齐的列输出数据。利用宽度限定可以轻松做到：

```c
#include <stdio.h>

int main(void)
{
    // 打印表头：- 号表示左对齐
    printf("%-6s | %-10s | %8s\n", "ID", "名称", "价格");
    printf("------+------------+----------\n");

    // 打印数据行
    printf("%-6d | %-10s | %8.2f\n", 1, "Apple", 5.50);
    printf("%-6d | %-10s | %8.2f\n", 2, "Banana", 3.20);
    printf("%-6d | %-10s | %8.2f\n", 3, "Cherry", 12.00);

    // 其中 %-12s 表示最少占 12 格，不够在后面补空格
    // %8.2f 表示最少占 8 格，小数点后 2 位，不足在前面补空格
    return 0;
}
```

### 1.3 控制浮点数显示精度

`printf` 可以精细控制浮点数的显示效果，这在输出金额、百分比等场景尤其有用：

```c
#include <stdio.h>

int main(void)
{
    double price = 114.514;

    // 保留 2 位小数（常见于金额）
    printf("金额: ¥%.2f\n", price);        // ¥114.51

    // 保留 6 位小数（默认精度）
    printf("金额: %f\n", price);            // 114.514000

    // 科学计数法
    printf("科学计数: %e\n", price);        // 1.145140e+02

    // 显示正号（强调符号）
    printf("%+d\n", 114);                   // +114
    printf("%+d\n", -514);                  // -514

    return 0;
}
```

### 1.4 百分号的输出技巧

输出 `%` 号需要用 `%%` 转义，这与 printf 的格式化机制有关：

```c
printf("完成进度: %d%%\n", 85);   // 完成进度: 85%
printf("税率: %.1f%%\n", 13.0);   // 税率: 13.0%
```

> ⚠️ 注意事项
>
> - 占位符数量与类型必须与参数一一对应，错误的匹配会导致未定义行为
> - `%s` 只能用于字符串指针，指针为 `NULL` 时可能导致崩溃，建议先检查
> - 宽度限定符 `%-` 左对齐在实践中非常常用，比纯文本拼接方案可读性高出很多

## 二、运算符实战技巧

### 2.1 整数除法的坑与应对

这是 C 语言中最容易出错的地方之一。整数 / 整数 = 整数，小数部分直接截断：

```c
#include <stdio.h>

int main(void)
{
    // 经典错误写法
    float a = 5 / 2;          // a = 2.0，不是 2.5！5/2 先算为 2，再转 float
    printf("5 / 2 = %f\n", a); // 2.000000 （用户预期 2.5）

    // 正确写法一：至少一方是浮点数
    printf("5.0 / 2 = %f\n", 5.0 / 2);      // 2.500000

    // 正确写法二：使用乘数扩展，最大程度避免截断误差
    int num = 5;
    num = (num * 100) / 20;   // 先乘 100 再除 20 = 25
    printf("(num*100)/20 = %d\n", num);

    // 对比：先除后乘——结果丢光
    num = 5;
    num = (num / 20) * 100;   // 5/20=0 → 0*100=0
    printf("(num/20)*100 = %d\n", num);  // 0

    return 0;
}
```

> **实战要诀**：如果计算比例，先乘后除，不要先除后乘。先除会丢失精度，而且丢了就再也回不来了。

### 2.2 自增自减的推荐写法

建议把自增/自减从复合表达式里拆出来单独一行，意图一目了然：

```c
#include <stdio.h>

int main(void)
{
    int i = 42;
    int j;

    // ❌ 容易混淆
    j = (i++ + 10);           // i 递增的时间点不易感知
    j = (++i + 10);           // 与上面行为不一样（差 1）

    // ✅ 推荐：分离操作
    i = 42;
    i++;
    j = i + 10;               // 不需停下来想 i 和 j 各是多少
    printf("j = %d\n", j);

    // ✅ 或者先计算，再加
    i = 42;
    j = i + 10;
    i++;
    printf("j = %d\n", j);

    return 0;
}
```

> **技巧**：将 `++`/`--` 单独成行，不嵌在表达式里，查看代码时不需要停下来心算。所有需要"停一步想一下"的东西都值得拆开。

### 2.3 `==` 与 `=` 的区分

这是 C 初学者每天踩一次的坑。

```c
#include <stdio.h>

int main(void)
{
    int x = 3;

    // ✅ 意中写法（== 比较）
    if (x == 3)
        printf("x is 3\n");

    // ❌ 常见笔误（= 赋值并返回 3，非零即为真，恒成立）
    if (x = 3)
        printf("This always prints!\n");

    // 防御性技巧：把常量放 `==` 前面。如果写错为 =，gcc 会报警
    if (3 == x)              // 写成 if (3 = x) 会直接报错
        printf("x is 3\n");

    return 0;
}
```

> `左常右变` 即把常量写在 `==` 左边——一旦写错成 `=` 就会产生编译错误。依赖编译器的保险费比依赖肉眼低得多。

### 2.4 逻辑短路求值的实战用途

```c
#include <stdio.h>

int main(void)
{
    // 技巧 1：安全的空指针检查
    // if (ptr != NULL && ptr->value > 0)
    //   当 ptr 为 NULL 时第二个条件不会执行，天然安全

    // 技巧 2：用 ! 判断变量是否为零
    int ready = 0;
    if (!ready)                      // 当 ready == 0 时进入
        printf("系统未就绪\n");

    // 技巧 3：用 !! 将任意值归一为 1 或 0
    int n = 114;
    int is_nonzero = !!n;            // is_nonzero = 1
    printf("!!114 = %d\n", is_nonzero);
    // 这是人为制造的布尔化写法，用于需要严格 0/1 的场合

    return 0;
}
```

> **技巧**：当 `!ready` 和 `!!data` 不足以解决所有问题时，可以手动写成 `x != 0 ? 1 : 0`，可读性更高。

### 2.5 位运算的实战场景

位运算不是概念体操，它们在以下场景中很常见。

```c
#include <stdio.h>

int main(void)
{
    // 场景 1：权限位检查（注意：0b 是 GNU 扩展 / C23 标准）
    unsigned int perms = 0b1101;     // 二进制位表示各个权限
    unsigned int mask   = 0b0100;    // 检查第 3 位

    if (perms & mask)               // 与运算验第 3 位是否有权限
        printf("指定权限通过\n");

    // 场景 2：置位（打开某个权限）
    perms |= mask;                   // |= 把指定位设为 1
    printf("打开第 3 位：%u\n", perms);

    // 场景 3：把第 N 位清零（去哪儿）
    perms &= ~mask;                  // 0b0111 & ~(0b1000)

    // 场景 4：位移用于打包/解包颜色分量
    // 快速乘除 2 的幂
    int value = 1;
    int doubled = value << 3;        // 1 * 2^3 = 8
    printf("1 << 3 = %d\n", doubled);

    // 高效除以 2 的幂（只有正数情才准确，负数看其是）
    int x = 32;
    int halved = x >> 1;             // 32 / 2 = 16
    printf("32 >> 1 = %d\n", halved);

    return 0;
}
```

位运算常用操作速查：

- `var & mask`：检查某一位是否为 1（与掩码进行位与）
- `var |= mask`：把某一位变成 1（与掩码位或）
- `var &= ~mask`：把某一位变成 0（与掩码取反后的位与）
- `var ^= mask`：翻转某一位（与掩码异或）
- `var << n`：快速 × 2^n（左移 n 位）
- `var >> n`：快速 ÷ 2^n（右移 n 位，正数适用）

> **进阶技巧**：用 `~0` 获取全 1 无符号数；同时用 `unsigned` 类型做位运算，因为 C 标准没有规定 `signed int` 右移时补 0 还是补 1。

### 2.6 运算优先级的实用建议

```c
#include <stdio.h>

int main(void)
{
    // 不用记忆 40+ 个优先级
    // 不确定的时候加圆括号，并且尽量分成多个语句

    // ❌ 真的需要这样写吗？
    int result = 3 + 4 * 5 >> 2 & 1;

    // ✅ 分成多步
    int step1 = 3 + 4 * 5;          // 乘法优先于加法还是加法优先？
                                     // 加了括号就清晰了
    int step2 = (step1 >> 2) & 1;   // 移位与位与的优先级一目了然
    printf("result = %d\n", step2);

    // 圆括号是最简单也最实惠的保险
    int x = (3 + 4) * 5;            // 35
    printf("(3+4)*5 = %d\n", x);

    return 0;
}
```

**行为准则**：
- 一会上关系/逻辑混合表达式直接加括号
- 一会上移位和加减乘除混合也加括号
- 需要多分几行就分几行，编译器不会罚你钱

## 三、条件判断实战

### 3.1 if 与 else if 的多条件判断

```c
#include <stdio.h>

int main(void)
{
    int score = 85;

    // 实现多级评分系统
    if (score >= 90)
        printf("评级：A\n");
    else if (score >= 80)
        printf("评级：B\n");
    else if (score >= 70)
        printf("评级：C\n");
    else if (score >= 60)
        printf("评级：D\n");
    else
        printf("评级：F\n");

    // 全过程：使用大括号 + 单一逻辑入口，减少重复赋值
    int max;
    int a = 42, b = 114;
    if (a > b) {
        max = a;
    } else {
        max = b;
    }
    printf("较大值是 %d\n", max);

    return 0;
}
```

**技巧**：`else if` 框架就是一个连续的筛选管线。利用它可以减少多层重复的条件判断，同时每一次 `else if` 都隐含“前面一切已被否定”。

### 3.2 防御式悬垂匹配——用括号消除歧义

```c
#include <stdio.h>

int main(void)
{
    int x = 114, imx = 42;

    // ❌ 看不清意图
    if (x > 110)
        if (imx > 0)
            printf("x > 110 and imx > 0\n");
    else
        printf("nothing\n");

    // 上面这个 else 其实是匹配内层 if (imx > 0)

    // ✅ 用括号把意图写出来
    if (x > 110) {
        if (imx > 0)
            printf("x > 110 and imx > 0\n");
    } else {
        printf("nothing\n");           // 这里 else 匹配外层的 x > 110
    }

    return 0;
}
```

**建议**：全部加括号，意图明确，不易出错。

### 3.3 三元运算符 `?:` 的实战用法

三元运算符的特点：它是一个**表达式**，可以直接用在赋值 `=` 右边。

```c
#include <stdio.h>

int main(void)
{
    int x = 1, y = 2;

    // 1. 用于赋值（单行二选一）
    int max = (x > y) ? x : y;
    printf("max = %d\n", max);

    // 2. 用于 printf 内（需要打印不同文本）
    printf("结果：%s\n", (max >= 10) ? "通过" : "不通过");

    // 3. 组合使用
    int result = (max >= 10) ? x : y;

    // 4. 复杂的就退化回普通 if/else 更好
    if (x > y) {
        printf("x is bigger\n");
        // ... 多步骤逻辑
    } else {
        printf("y is >= x\n");
        // ... 多步骤逻辑
    }

    return 0;
}
```

**朴素建议**：
- 简单二选一用 `?:`，保持紧凑
- 一旦有两个以上判断动作就别用三元；用 if 更清晰
- 别嵌套三元（比如 `a ? b : c ? d : e`），那是给以后维护的人埋雷

### 3.4 switch 的实战用法

`switch` 主跑多分支场景（菜单、状态机、命令分发）：

```c
#include <stdio.h>

int main(void)
{
    int command = 2;

    switch (command) {
    case 1:
        printf("执行新建操作\n");
        break;
    case 2:
        printf("执行删除操作\n");
        break;
    case 3:
        printf("执行复制操作\n");
        break;
    case 4:
        printf("执行移动操作\n");
        break;
    default:
        printf("无效命令\n");
    }

    // 利用穿透——多个值共享一份逻辑
    char op = '+';
    switch (op) {
    case '+':
    case '-':
    case '*':
    case '/':
        printf("是算术运算符\n");
        break;
    default:
        printf("未知操作符\n");
    }

    return 0;
}
```

**本报提醒**：
- `default` 必须存在，即使你的逻辑从理论上不应该走到那，也是一种保险
- 每个 case 的 `break` 不要忘，否则穿透继续执行下一个 case 的逻辑体
- case 穿透不是 bug，是有意的 yield 语义——不写 break 就表示“流到下一个 case”

## 四、循环实战

### 4.1 选 while 还是 for？

```c
#include <stdio.h>

int main(void)
{
    // 知道运行多少次 → for
    for (int i = 0; i < 5; i++) {
        printf("第 %d 遍运行\n", i + 1);
    }

    // 不知道运行次数、循环终止在某种外部条件（读文件，等输入） → while
    int ch;
    while ((ch = getchar()) != '\n' && ch != EOF) {
        // 处理字符
    }

    return 0;
}
```

> 如果循环的退出条件**不依赖于计数**、更多依赖于外部的数据源或者状态变化，用 `while` 会比 `for` 更自然。

### 4.2 do...while 保证至少执行一次

当必须至少执行一次操作才判断是否继续时（如菜单循环提示）：

```c
#include <stdio.h>

int main(void)
{
    int x;
    x = 20;

    // 至少执行一次 i--（即便 x 已经是 0）
    do {
        x--;
        printf("x = %d\n", x);
    } while (x > 0);    // 别忘记这里的分号

    // 条件一开始就为假，循环体也执行了一次
    x = 0;
    do {
        printf("这段话还是会输出一次\n");
    } while (x > 0);

    return 0;
}
```

注意最后的 `;` 分号不能省略！

### 4.3 无限循环结构

```c
#include <stdio.h>

int main(void)
{
    // 写法一：for 无限循环（最紧凑）
    for (;;) {
        printf("Tick \n");
        // sleep(1);   // 实际必须有一个退出条件
        break;         // 该 break 为了示例安全性
    }

    // 写法二：while 无限循环
    while (1) {
        printf("Tock \n");
        break;
    }

    return 0;
}
```

**对于任何无限循环**，内部必须有 `break`、`return`、超时，或者信号触发的退出路径。

### 4.4 嵌套循环的 break 与 continue

```c
#include <stdio.h>

int main(void)
{
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 3; j++) {
            if (j == 1) continue;   // 跳过 j == 1 的打印
            printf("i=%d, j=%d\n", i, j);
        }
    }

    printf("--- break ---\n");

    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 3; j++) {
            if (j == 1) break;      // 只跳出内层 j 循环
            printf("i=%d, j=%d\n", i, j);
        }
    }

    return 0;
}
```

`break` / `continue` 只影响**距离它们最近的那一层**循环，不会影响外层循环。

### 4.5 输入解析循环（getchar 实战）

```c
#include <stdio.h>

int main(void)
{
    int ch;

    printf("请输入字符：");
    // 读用户输入的每个字符，遇到回车或 EOF 停止
    while ((ch = getchar()) != '\n' && ch != EOF) {
        if (ch == '\t') continue;          // 忽略制表符
        if (ch == ' ') {
            printf("_");                   // 把空格用 '_' 打印出来
        } else {
            putchar(ch);
        }
    }
    printf("< EOL\n");  // 证明已经到行末

    return 0;
}
```

**EOF 触发说明**：
- 在 Unix/Linux 终端下，按 `Ctrl+D` 发送 EOF
- 如果当前缓冲区有未提交的数据，第一次 `Ctrl+D` 会提交缓冲区内容（等效回车但不含换行符），第二次 `Ctrl+D` 才会真正发送 EOF

## 五、goto 的适用场景

`goto` 被普遍认为破坏结构化编程，以下场景是其实际维护中为数不多的合理用法：

### 5.1 深层循环出圈

在多层循环中，`break` 只能退出一层。`goto` 能一步退深井：

```c
// 伪代码，示意结构
for (int i = 0; i < ROWS; i++) {
    for (int j = 0; j < COLS; j++) {
        while (some_condition(i, j)) {
            do {
                if (error_condition())
                    goto bail;        // 一步跳出所有循环
            } while (condition);
        }
    }
}
// 统一清理
cleanup:
    printf("在 runaway 处退出\n");
```

### 5.2 连续错误检查

连续处理多个可能失败的操作时，可以用 `goto` 集中做错误处理：

```c
#include <stdio.h>

int do_sth1() { return 0; }  // 0 表示成功
int do_sth2() { return 0; }

int main(void)
{
    if (do_sth1() != 0) goto error;
    if (do_sth2() != 0) goto error;

    printf("所有操作成功\n");
    return 0;

error:
    printf("有操作失败，停止处理\n");
    // 在此统一清理或回滚
    return 1;
}
```

相比多个嵌套 if，此模式能极大降低代码嵌套深度。

## 六、常用编码风格与总结

### 6.1 大括号之争

虽然 C 允许单语句不加大括号，但从实践角度出发，始终加上大括号更安全：

```c
// ✅ 意图明确，易于长期维护
if (x > 0)
{
    x--;
}
else
{
    x++;
}
```

### 6.2 变量声明放到循环附近

```c
// C23/C99：在 for 语句内声明变量
for (int i = 0; i < 10; i++)
{
    // ...
}
// i 无法在此访问（作用域仅限于循环体内）

// 如果需要在循环结束后使用循环变量——在 for 外单独声明
int i;
for (i = 0; i < 10; i++)
{
    // ...
}
printf("循环结束于 %d\n", i);  // i 可以直接访问
```

### 6.3 多层循环配合 break/continue 的排错思路

当 break/continue/goto 混在多层循环里时，建议按以下步骤排查：

1. 在最内层放置 `printf`，确认每轮循环的执行情况
2. 用注释标注每个 `break`/`continue` 的作用范围
3. 将复杂循环提取为 `while` 形式，获得更高的可读性
4. 记住：`break` 只退出最内层的 `for`/`while`/`switch`

> ⚠️ 注意事项
>
> - **配对检查**：每个 `else` 只配最近的 `if`；每个 `break`/`continue` 仅影响最内层循环
> - **多用大括号**：省略大括号省的是打字时间，赔的是调试时间
> - **多用圆括号**：信任编译器，但更需要表达式意图清晰
> - **优先级算不清就拆多步**：拆成多个命名变量比一行塞几十个符号更值
> - **多用中间变量**：复杂表达式拆分为多个有意义的变量名，让代码自文档化
