# C 语言流程控制参考手册

## 一、if 语句

`if` 语句用于条件判断，当条件满足时执行指定的语句。

语法：

```c
if (test) run;
```

- 判断条件 `test` 放在圆括号内
- 当 `test` 判断为真（非零）时，执行后面的语句体

语句体可以是单个语句，也可以是放在大括号里的复合语句：

```c
int x = 114;
if (x == 114)
    printf("x is 114\n");        // 单语句，通常另起一行加缩进
```

```c
int imx = INT_MAX;
if (imx == INT_MAX)
{
    printf("imx is %i\n", imx);  // 复合语句放在大括号内
    imx = 0;
    imx++;
}
```

### if else 分支

`if` 有一个 `else` 分支，指定当条件判断为伪时执行的命令：

```c
if (test)
    run;
else
    run;
```

示例：

```c
if (imx > x)
    printf("imx is bigger than x\n");
else
    printf("imx is smaller than x\n");
```

`else` 后面同样可以是单个语句或大括号内的复合语句：

```c
int max;
if (x > imx)
{
    max = x;
    printf("max is %i\n", max);
} else {
    max = imx;
    printf("max is %i\n", max);
}
```

### 嵌套 if（else if 链）

`if` 可以嵌套串联，构成多条件判断链：

```c
int a = 1, b = 2;
if (a == 1)
    printf("a is 1\n");
else if (b == 2)
    printf("b is 2\n");
```

上面先判断 `a` 是否为 1，如果是则执行第一条分支并结束；如果不是，再判断 `b` 是否为 2。这种结构可用于 fallback（多级回退）场景。

### else 的悬垂匹配

`else` 只跟与它最近的一个 `if` 匹配。「最近」指的是编译时在时间上与该 `else` 相差最短的 `if`，与之后的 `if` 无关。

```c
if (x > 110)
    if (imx > 0)
        printf("x > 110 and imx > 0\n");
else
    printf("nothing\n");
```

上面的缩进容易让人以为 `else` 匹配第一个 `if`（`x > 110`），但实际上 `else` 匹配的是第二个 `if`（`imx > 0`）。

实际执行逻辑：

- `x > 110` 为真时，进入内层 `if`
  - `imx > 0` 为真 → 输出 `"x > 110 and imx > 0"`
  - `imx > 0` 为伪 → 输出 `"nothing"`
- `x > 110` 为伪 → 什么都不做

> **⚠️ 注意事项**
>
> 1. **else 悬垂问题**：`else` 总是匹配最近的未配对 `if`，缩进不能改变这个规则。写法不明确时容易让维护者理解错意图，甚至让机器理解错意图。
>
> 2. **推荐显式使用大括号**：为了明确意图、便于阅读、防止出错，建议无论单语句还是复合语句都加上大括号：
>
> ```c
> if (x > 210)
> {
>     if (imx > 0)
>     {
>         printf("x is bigger than 110 and imx is bigger than 0\n");
>     }
> } else
> {
>     printf("nothing\n");
> }
> ```
>
> 大括号明确了哪个 `if` 属于哪个 `else`，消除了歧义，也使代码更易读。

## 二、switch 语句

`switch` 语句是一种特殊形式的 `if...else` 结构，用于判断条件有多个结果的情况。它把多重的 `else if` 改成更易用、可读性更好的形式。

基础语法：

```c
switch (expression)
{
    case value1 : statement
    case value2 : statement
    default : statement
}
```

根据表达式 `expression` 不同的值，执行相应的 `case` 分支。如果找不到对应的值，就执行 `default` 分支。

示例：

```c
int x;
x = 114;
switch (x)
{
    case 1 :
        printf("true\n");
        break;
    case 0 :
        printf("false\n");
        break;
    default :
        printf("no true or false\n");
}
```

根据变量 `x` 不同的值，会执行不同的 `case` 分支。如果等于 0，执行 `case 0` 的部分；如果等于 1，执行 `case 1` 的部分；否则，执行 `default` 的部分。`default` 表示处理以上所有 case 都不匹配的情况。

### break 语句与 case 穿透

每个 `case` 语句体的结尾，都应该有一个 `break` 语句，作用是跳出整个 `switch` 结构，不再往下执行。如果缺少 `break`，就会导致继续执行下一个 `case` 或 `default` 分支。

```c
switch (x)
{
    case 0 :
    case 1 :
        printf("true\n");
        break;
    default :
        printf("Illegal\n");
}
```

利用这一特性，可以让不同结果执行相同语句——上面示例中，`x` 为 0 或 1 时会穿透到同一个 `printf("true\n")`。

`case` 后面的语句体不用放在大括号里面，这也是为什么需要 `break` 的原因。

### default 分支

`default` 分支用来处理前面的 `case` 都不匹配的情况，最好放在所有 `case` 的后面，这样就不用写 `break` 语句。这个分支是可选的——如果没有该分支，遇到所有的 `case` 都不匹配的情况，就会直接跳出整个 `switch` 代码块。

> **⚠️ 注意事项**
>
> 1. **switch 通常不用来做真假判断**，而是用于条件选择。当变量取值为 `0`、`1` 时，编译器可能会警告——这种场景更适合用 `if` 语句。
>
> 2. **不要忘记 `break`**：每个 `case` 分支结尾需要 `break`，否则会穿透执行下一个分支。这是新手最容易犯的错误。
>
> 3. **case 穿透是特性而非 bug**：多个 `case` 共用一个语句体时，可以省略前几个 `case` 的 `break`，让它们"穿透"到同一个处理逻辑。善用这一特性可以提高代码的简洁性。
>
> 4. **`default` 放在最后**：将 `default` 放在所有 `case` 后面是最佳实践，此时不需要写 `break`。
>
> 5. **`case` 后不用大括号**：`case` 后面的语句体不需要放在大括号里，`break` 就是边界。

## 三、三元运算符 `?:`

C 语言存在一个三元表达式 `?:`，可以用作 `if...else` 的简写形式。

语法：

```c
<expression1> ? <expression2> : <expression3>
```

含义：表达式 `expression1` 如果为 true（非零值），就执行 `expression2`，否则执行 `expression3`。

示例：

```c
int x;
x = 1;
x == 1 ? printf("x is 1\n") : printf("x is %i", x);  // 语句不添加分号

int y;
y = 2;
(x > y) ? printf("x is bigger than y\n") : printf("y is >= x\n");  // 判断大小并输出
```

三元运算符也可以用于返回值而不输出：

```c
(x > y) ? x : y;  // 判断大小并返回值，等价于：

if (x > y)
    return x;
else
    return y;
```

> **⚠️ 注意事项**
>
> 1. **三元运算符的语句不添加分号**：`expression2` 和 `expression3` 后面不需要分号，分号加在整个三元表达式末尾。
>
> 2. **三元运算符是表达式**，有返回值（返回 `expression2` 或 `expression3` 的值），因此可以直接用于赋值或 `return`。
>
> 3. **可读性权衡**：三元运算符适合简单的二选一场景，可以让代码更紧凑。但如果条件或分支逻辑过于复杂，建议使用完整的 `if...else` 结构以提高可读性。

## 四、while 语句

`while` 语句用于循环结构，满足条件时，不断执行循环体。

标准格式：

```c
while (expression)
    statement
```

例如：

```c
while (i < n)
    i + 2;
```

上面示例中，只要 `i` 小于 `n`，`i` 就会不断增加 2。

循环体可以是复合语句（放在大括号中）：

```c
while (expression)
{
    statement;
    statement;
}
```

一个完整示例：

```c
int x;
x = 0;
while (x < 10)
{
    printf("x is %i now\n", x);
    x++;
}  // 循环 10 次
```

### 无限循环

只要条件为真，`while` 会产生无限循环。下面是一种常见的无限循环的写法：

```c
while (n)
{
    // ...
}
```

上面示例虽然是无限循环，但是循环体内部可以用 `break` 语句跳出循环。

> **⚠️ 注意事项**
>
> 1. **确保循环条件最终会变为伪**：除非有意写无限循环，否则循环体内应该有改变条件变量的逻辑（如 `x++`），避免死循环。
>
> 2. **无限循环不是 bug**：`while (1)` 或 `while (n)`（n 恒为非零）是合法的无限循环写法，常用于服务器主循环、事件监听等场景。配合 `break` 语句可以在适当时机退出。
>
> 3. **循环体可以是单语句或复合语句**：单语句直接跟在条件后，复合语句用大括号包裹。建议始终使用大括号，即使只有一条语句，以保持一致性并防止后续修改时出错。

## 五、for 语句

`for` 语句是 C 语言中最常用的循环结构，通常用于精确控制循环次数。

语法：

```c
for (initialization; continuation; action)
    statement;
```

圆括号内有三个表达式，用分号分隔：

- **initialization（初始化表达式）**：用于初始化循环变量，只在循环开始前执行一次
- **continuation（判断表达式）**：每轮循环开始前求值，只要为真（非零），就执行循环体
- **action（循环变量处理表达式）**：每轮循环体执行结束后求值，通常用于更新循环变量

例如：

```c
for (int x = 10; x > 5; x--)
    printf("x is %i\n", x);
```

上面示例中，`x` 初始化为 10，只要 `x > 5` 就打印 `x` 的值，每轮结束后 `x` 自减 1。共输出 5 行（10、9、8、7、6）。

### 多个表达式

`for` 语句的初始化和处理部分都可以包含多个表达式，表达式之间用逗号分隔：

```c
int i, j;
for (i = 10, j = 999; i < 10; i++, j--)
{
    printf("i is %d now and j is %d now\n", i, j);
}
```

上面示例中，初始化部分同时给 `i` 和 `j` 赋值，处理部分同时让 `i` 自增、`j` 自减。

### 省略表达式

`for` 语句的三个表达式都不是必需的，可以部分或全部省略（但分号不能省略）。以下是一种无限循环的写法：

```c
for (;;)
{
    printf("not limit\n");
    sleep(1);  // 每秒输出一次，永远不会停止
}
```

全部省略三个表达式时，判断条件默认为真，循环将无限执行。

> **⚠️ 注意事项**
>
> 1. **三个表达式用分号分隔，不是逗号**：`for` 圆括号内用两个分号分隔三个表达式，初始化或处理内部有多个子表达式时才用逗号。
>
> 2. **循环变量作用域**：在初始化表达式中声明的变量（如 `for (int x = 0; ...)`）的作用域仅限于 `for` 循环体内，循环结束后不可访问。如果需要循环结束后继续使用，应在 `for` 之前声明。
>
> 3. **分号不能省略**：即使三个表达式全部省略，两个分号也必须保留，写成 `for (;;)`。
>
> 4. **省略判断条件默认为真**：省略第二个表达式时，循环条件恒为真，变成无限循环。除非循环体内有 `break`，否则程序不会停止。
>
> 5. **sleep 参数为无符号整数**：C 语言的 `sleep()` 函数参数类型为 `unsigned int`，传入浮点数（如 `1.2`）会被截断为整数（如 `1`）。

## 六、break 语句

`break` 语句有两种用法：

- **在 `switch` 语句中**：中断某个 `case` 分支的执行，跳出整个 `switch` 结构（前面已介绍）
- **在循环体内部**：立即跳出当前循环，不再执行后续迭代

### 在循环中使用 break

`break` 只跳出它所在的**最内层**循环。以下示例演示了嵌套循环中 `break` 的行为：

```c
for (int i = 0; i < 3; i++)
{
    for (int j = 0; j < 3; j++)
    {
        printf("j is %d now\n", j);
        break;
    }
}
```

上面循环中，内层 `for` 每次进入后只执行一次（`j = 0` 时），因为 `break` 立即跳出内层循环。程序实际行为：

- `i = 0`：进入内层，`j` 从 0 开始，打印一次后 `break` 跳出内层 → 回到外层，`i++`
- `i = 1`：重新进入内层，`j` 又从 0 开始，打印一次后 `break` 跳出 → 回到外层，`i++`
- `i = 2`：同上

`j++` 永远不会被执行——每次进入内层循环时 `j` 都是 0，打印后就 `break`，没有机会自增。如果去掉 `break`，`j` 会正常从 0 循环到 2，然后 `i++` 之后再重新执行内层。

### 配合 while 提前退出

`break` 常用于在满足某个条件时提前终止循环：

```c
int ch;
while ((ch = getchar()) != EOF)
{
    if (ch == '\n') break;  // 读到换行符就跳出
    putchar(ch);
    printf("\n");
}
```

上面循环中：

- `getchar()` 从标准输入读取单个字符
- 判断是否为 `EOF`（文件结束符），是则终止循环
- 判断是否为换行符 `\n`，是则 `break` 跳出循环
- 否则打印该字符并继续循环

### break 的作用范围

`break` 只能跳出**循环体**和 **`switch` 结构**，不能跳出 `if` 语句：

```c
int n = 3;
if (n > 1)
{
    if (n > 2)
    {
        // break; // 编译报错！break 不能用于跳出 if
        printf("Hello World\n");
    }
}
```

在 `if` 中使用 `break` 会导致编译错误。

> **⚠️ 注意事项**
>
> 1. **break 只跳出最内层循环**：嵌套循环中，`break` 只跳出直接包含它的那一层循环，外层循环不受影响。
>
> 2. **break 不能用于 if**：`break` 只能跳出循环体或 `switch` 结构，放在 `if` 语句中而没有外层循环/`switch` 包裹会导致编译错误。
>
> 3. **EOF 的触发方式**：在终端输入时，按 `Ctrl+D`（Unix/Linux）可以发送 `EOF`。注意：如果当前行有未提交的输入，第一次 `Ctrl+D` 会提交当前输入（等效回车但不含换行符），需要再按一次才能发送 `EOF`。
>
> 4. **getchar 的缓冲机制**：`getchar()` 每次只读取一个字符，但终端输入是行缓冲的——用户在按回车前输入的所有字符都暂存在缓冲区中。因此循环会逐个消耗缓冲区中的字符，直到遇到换行符或 `EOF`。

## 七、continue 语句

`continue` 语句用于在循环体内部终止本轮循环，进入下一轮循环。只要遇到 `continue` 语句，循环体内部后面的语句就不执行了，回到循环体的头部，开始执行下一轮循环。

### 在 for 循环中使用 continue

在 `for` 循环中，`continue` 会跳过本轮剩余的语句，跳转到"循环变量处理表达式"（`action`）部分，然后再进行条件判断：

```c
for (int i = 10; i > 0; i--)
{
    for (int j = 10; j > 0; j--)
    {
        printf("j is %d now\n", j);
        continue;
    }
}
```

上面示例中，有没有 `continue` 语句效果一样——因为 `continue` 后面已经没有其他语句了，`continue` 无非是跳到下一个 `j`（即执行 `j--` 然后判断）。

### 在 while 循环中使用 continue 跳过字符

`continue` 常用于在循环中跳过某些特定条件，提前进入下一轮：

```c
int ch;
printf("Please type a character: ");
while ((ch = getchar()) != '\n' && ch != EOF)
{
    if (ch == '\t') continue;
    putchar(ch);
}
```

上面示例中，只要读到的字符是制表符（`\t`），就用 `continue` 语句跳过 `putchar(ch)`，直接读取下一个字符。这样制表符就不会被打出来。

> **⚠️ 注意事项**
>
> 1. **continue 与 break 的区别**：
>    - `break`：立即退出**整个循环**，不再执行任何后续迭代
>    - `continue`：只退出**本轮循环**，继续执行下一轮迭代
>
> 2. **continue 在 while 中的位置要注意**：在 `while` 循环中，`continue` 会跳转到条件判断部分，而不是循环变量更新。如果在 `continue` 前没有更新循环变量，可能导致死循环：
>
> ```c
> // 有问题：continue 跳过了 i++，可能导致死循环
> while (i < 10)
> {
>     if (i == 5) continue;  // 跳到 while 条件判断，i++ 被跳过
>     printf("%d\n", i);
>     i++;
> }
>
> // 而在 for 中则安全，因为 action 表达式（如 i++）每轮都会执行
> for (int i = 0; i < 10; i++)
> {
>     if (i == 5) continue;  // 跳过后仍会执行 i++
>     printf("%d\n", i);
> }
> ```
>
> 3. **continue 只能用于循环体内部**：和 `break` 类似，`continue` 只能出现在循环体中（`for`、`while`、`do...while`），放在循环外或 `if` 中没有意义。

## 八、goto 语句

`goto` 语句用于无条件跳转到指定的标签名。这会破坏结构化编程，**建议不要轻易使用**——这里为了语法的完整性，介绍它的用法。

### 基本语法

`goto` 语句有两个部分：

- **标签（Label）**：写在某条正常语句的前面，形式为 `标签名 : 语句`，相当于为该行语句做了一个标记
- **goto 语句**：`goto 标签名;`，程序执行到 `goto` 就会跳转到对应标签的位置

```c
char ch;
top : ch = getchar();

if (ch == 'q')
    goto top;
```

上面示例中，`top` 是一个标签名，程序执行到 `goto top` 时就会跳转到 `top :` 所在的位置。在这个示例中，如果输入了 `q`，就重新读一个新的字符，相当于忽略了字符 `q`。

### goto 实现无限循环

标签可以搭配 `goto` 形成无限循环：

```c
infinite_loop :
    printf("Hello World\n");
    sleep(1);
    goto infinite_loop;
```

上面代码会不停地打印 "Hello World"，每秒一次，永远不会停止。

### goto 的主要用法

虽然 `goto` 被广泛认为是有害的，但在某些场景下它仍是合理的解决方法。

**（1）跳出多层循环**

当普通 `break` 只能跳出最内层循环时，如果在深层嵌套中检测到错误，`goto` 可以一次性跳出所有层：

```c
for (...)
{
    for (...)
    {
        while (...)
        {
            do {
                if (something_error_condition)
                    goto bail;
            } while (...);
        }
    }
}

bail:
// 错误处理逻辑...
```

在这个场景中，`break` 需要逐层退出（每一层都要加判断），代码会变得非常臃肿。`goto` 直接跳到统一出口，简洁明了。

**提早结束多重判断**

在连续进行多个条件判断时，只要有一个发现错误，就可以用 `goto` 跳过后面的判断：

```c
if (do_sth1() == ERR)
    goto error;
if (do_sth2() == ERR)
    goto error;
if (do_sth3() == ERR)
    goto error;

error:
// 错误处理逻辑...
```

上面示例中，三个操作依次执行。只要其中一个返回错误，就跳转到统一的 `error` 处理标签，不再执行后续的操作。这种模式在需要执行多个可能失败的操作时很常见。

> ⚠️ 注意事项
>
> 1. **不要轻易使用 `goto`**：`goto` 会破坏结构化编程，让代码的执行流程变得难以追踪，增加调试和维护的难度。除非确实需要（如跳出深层循环），否则优先使用 `if`、`for`、`while` 等结构化语句。
>
> 2. **`goto` 只能在同一个函数之中跳转**：不能从一个函数跳转到另一个函数，标签名的作用域仅限于当前函数。
>
> 3. **标签名可以放在任何语句的前面**：标签只是一个标记点，不影响该语句的正常执行。如果不加 `goto` 跳转，程序顺序执行时会直接经过标签，就像它不存在一样。
>
> 4. **避免用 `goto` 向前跳**：优先让 `goto` 向后跳（跳到下面错误处理之类的区域），而不是向前跳形成回环（回环应该用循环结构）。若 `goto` 向前跳，很轻易就可以把代码搞成一团乱麻。
