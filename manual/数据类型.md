# C 语言数据类型参考手册

## 一、数据类型概述

C 语言的每一种数据都有类型（type），编译器必须知道数据的类型才能操作数据。所谓"类型"，就是相似的数据所拥有的共同特征——一旦知道某个值的数据类型，就能知道该值的特征和操作方式。

基本数据类型有三种：

- `char`：字符类型
- `int`：整数类型
- `float`：浮点数类型

复杂的类型都是基于这三种基本类型构建的。

> ⚠️ 注意事项
>
> - 类型决定了数据的存储大小、取值范围和可执行的操作。不同类型之间转换时要留意数据丢失或截断的风险（后续章节会详述）。

## 二、字符类型

字符类型指单个字符，类型声明使用 `char` 关键字。

```c
char a = 'B';
```

上面声明了变量 `a` 是字符类型并赋值为 `B`。

### 字符常量与单引号

C 语言规定，字符常量必须放进单引号（`'`）里面。单引号本身也是一个字符，如果要表示这个字符常量，必须使用反斜杠转义：

```c
char t = '\'';
```

上面变量 `t` 为单引号字符。由于字符常量必须放在单引号里面，所以内部的单引号要使用反斜杠转义。

### char 的底层存储

在计算机内部，字符类型使用一个字节（8 bit）存储。C 语言将其当作整数处理，所以字符类型就是宽度为一个字节的整数。每个字符对应一个整数（由 ASCII 码确定），比如 `B` 对应整数 66。

字符类型在不同计算机上的默认范围是不一样的：

- 一些系统默认为 -128 到 127
- 另一些系统默认为 0 到 255

这两种范围正好都能覆盖 0 到 127 的 ASCII 字符范围。

### 字符与整数互换

只要在字符类型的范围之内，整数与字符是可以互换的——都可以赋值给字符类型的变量：

```c
char c = 66;
printf("%c\n", c);   // 用 %c 打印字符 → B

c = 'B';
printf("%c\n", c);   // 同上，也是 B

// 两种写法完全等价
char a = 'B';        // 等同 a = 66;
char b = 'C';        // 等同 b = 67;
printf("%i\n", a + b);  // 字符可参与整数运算 → 133
```

上面示例中，变量 `c` 是字符类型，赋值为整数 66 跟赋值为字符 `B` 的效果是一样的。字符类型本质上就是整数，可以直接参与算术运算。

### 转义字符

转义写法主要用来表示 ASCII 码定义的一些无法打印的控制字符，它们也属于字符类型的值。

控制字符：

- `\a`：警报，使终端发出警报声或闪烁，或两者同时发生
- `\b`：退格键，光标回退一个字符，但不删除字符
- `\f`：换页符，光标移到下一页。在现代系统上，这已经反映不出来了，行为改成类似于 `\v`
- `\n`：换行符
- `\r`：回车符，光标移到同一行的开头
- `\t`：制表符，光标移到下一个水平制表位，通常是下一个 8 的倍数
- `\v`：垂直分隔符，光标移到下一个垂直制表位，通常是下一行的同一列
- `\0`：null 字符，代表没有内容。注意，这个值不等于数字 0

八进制和十六进制写法：

转义写法还能使用八进制和十六进制表示一个字符。

- `\nn`：字符的八进制写法，`nn` 为八进制值
- `\xnn`：字符的十六进制写法，`nn` 为十六进制值

以下四种写法完全等价：

```c
char x = 'B';
x = 66;          // 十进制整数
x = '\102';      // 八进制转义（'B' 的 ASCII 码 66 的八进制是 102）
x = '\x42';      // 十六进制转义（66 的十六进制是 42）
```

> ⚠️ 注意事项
>
> 1. **字符常量必须用单引号**：`'A'` 是字符，`"A"` 是字符串（两者不同，字符串包含末尾的 `\0`）。不能用双引号包单个字符。
>
> 2. **char 是有符号还是无符号取决于系统和编译器**：范围为 -128~127 时，大于 127 的值会溢出并回绕到负数。如果需要存储 0~255 范围的值，可以使用 `unsigned char`。
>
> 3. **`\0` 不等于数字 `0`**：`\0` 是 ASCII 码值为 0 的字符（null 字符），常用于标记字符串结尾。数字 `0` 的 ASCII 码值是 48（即字符 `'0'`）。
>
> 4. **转义字符是单个字符**：虽然 `\n`、`\t`、`\x42` 写成了两个或多个字符的形式，但在编译后它们都只占一个字节。不要把它们和字符串混淆。
>
> 5. **控制字符在现代终端中的行为可能不统一**：比如 `\a` 在某些终端上不发出声音，`\f` 的行为在现代系统上已退化。学习时了解即可，实际开发中主要使用 `\n`、`\t`、`\0` 这几个。

## 三、整数类型

整数类型用来表示较大的整数，类型声明使用 `int` 关键字。

```c
int a;
```

上面用整数类型声明了一个整数变量 `a`。

### int 的大小与范围

不同计算机的 `int` 类型的大小是不一样的。比较常见的是使用 4 个字节（32 位）存储一个 `int` 类型的值，但是 2 个字节（16 位）或 8 个字节（64 位）也有可能使用。它们可以表示的整数范围如下：

- 16 位：-32,768 到 32,767
- 32 位：-2,147,483,648 到 2,147,483,647
- 64 位：-9,223,372,036,854,775,808 到 9,223,372,036,854,775,807

```c
a = INT_MAX;
printf("%d\n", a);
```

### signed 与 unsigned

C 语言使用 `signed` 关键字，表示一个类型带有正负号，包含负值；使用 `unsigned` 关键字，表示该类型不带有正负号，只能表示零和正整数。

对于 `int` 类型，默认是带有正负号的，也就是说 `int` 等同于 `signed int`。由于这是默认情况，关键字 `signed` 一般都省略不写，但是写了也不算错。

```c
signed int b; // 等同于 int b;
```

`int` 类型也可以不带正负号，只表示非负整数，这时就必须使用关键字 `unsigned` 声明变量：

```c
unsigned int c = 4294967294;
```

整数变量声明为 `unsigned` 的好处是，同样长度的内存能够表示的最大整数值增大了一倍。比如，16 位的 `signed int` 最大值为 32,767，而 `unsigned int` 的最大值增大到了 65,535。

`unsigned int` 里面的 `int` 可以省略，所以上面的变量声明也可以写成：

```c
unsigned a;
```

字符类型 `char` 也可以设置 `signed` 和 `unsigned`：

```c
signed char c;   // 范围为 -128 到 127
unsigned char c; // 范围为 0 到 255
```

注意，C 语言规定 `char` 类型默认是否带有正负号，由当前系统决定。这就是说，`char` 不等同于 `signed char`，它有可能是 `signed char`，也有可能是 `unsigned char`。这一点与 `int` 不同，`int` 就是等同于 `signed int`。

### 整数的子类型

如果 `int` 类型使用 4 个或 8 个字节表示一个整数，对于小整数，这样做很浪费空间。另一方面，某些场合需要更大的整数，8 个字节还不够。为了解决这些问题，C 语言在 `int` 类型之外，又提供了三个整数的子类型：

- `short int`（简写为 `short`）：占用空间不多于 `int`，一般占用 2 个字节（整数范围为 -32,768~32,767）
- `long int`（简写为 `long`）：占用空间不少于 `int`，至少为 4 个字节
- `long long int`（简写为 `long long`）：占用空间多于 `long`，至少为 8 个字节

```c
short int a;
long int b;
long long int c;
```

上面代码分别声明了三种整数子类型的变量。

默认情况下，`short`、`long`、`long long` 都是带符号的（`signed`），即 `signed` 关键字省略了。它们也可以声明为不带符号（`unsigned`），使得能够表示的最大值扩大一倍：

```c
unsigned short int a;
unsigned long int b;
unsigned long long int c;
```

C 语言允许省略 `int`，所以变量声明语句也可以写成下面这样：

```c
short a;
unsigned short a;
long a;
unsigned long a;
long long a;
unsigned long long a;
```

不同的计算机，数据类型的字节长度是不一样的。确实需要 32 位整数时，应使用 `long` 类型而不是 `int` 类型，可以确保不少于 4 个字节；确实需要 64 位的整数时，应使用 `long long` 类型，可以确保不少于 8 个字节。另一方面，为了节省空间，只需要 16 位整数时，应使用 `short` 类型；需要 8 位整数时，应使用 `char` 类型。

### 整数类型的极限值

有时需要查看当前系统不同整数类型的最大值和最小值，C 语言的 `<limits.h>` 头文件提供了相应的常量。为了代码的可移植性，需要知道某种整数类型的极限值时，应该尽量使用这些常量：

- `SCHAR_MIN`，`SCHAR_MAX`：`signed char` 的最小值和最大值
- `SHRT_MIN`，`SHRT_MAX`：`short` 的最小值和最大值
- `INT_MIN`，`INT_MAX`：`int` 的最小值和最大值
- `LONG_MIN`，`LONG_MAX`：`long` 的最小值和最大值
- `LLONG_MIN`，`LLONG_MAX`：`long long` 的最小值和最大值
- `UCHAR_MAX`：`unsigned char` 的最大值
- `USHRT_MAX`：`unsigned short` 的最大值
- `UINT_MAX`：`unsigned int` 的最大值
- `ULONG_MAX`：`unsigned long` 的最大值
- `ULLONG_MAX`：`unsigned long long` 的最大值

```c
#include <limits.h>

int a = INT_MAX;
printf("%d\n", a);  // 输出当前系统 int 的最大值
```

### 整数的进制

C 语言的整数默认都是十进制数。如果要表示八进制数和十六进制数，必须使用专门的表示法：

- 八进制：使用 `0` 作为前缀，比如 `017`、`0377`
- 十六进制：使用 `0x` 作为前缀，比如 `0x1A2B`（相当于 6699）
- 二进制（C23 起为标准，此前为 GNU 扩展）：使用 `0b` 作为前缀，比如 `0b101010`

```c
int octal = 012;       // 八进制，十进制为 10
int hex = 0x1A2B;      // 十六进制，十进制为 6699
int bin = 0b101010;    // 二进制（C23），十进制为 42
```

注意，不同的进制只是整数的书写方法，不会对整数的实际存储方式产生影响。所有整数都是二进制形式存储，跟书写方式无关。不同进制可以混合使用，比如 `10 + 015 + 0x20` 是一个合法的表达式。

`printf()` 相关的进制占位符：

- `%d`：十进制整数
- `%o`：八进制整数
- `%x`：十六进制整数（小写）
- `%#o`：带 `0` 前缀的八进制整数
- `%#x`：带 `0x` 前缀的十六进制整数
- `%#X`：带 `0X` 前缀的十六进制整数

```c
printf("%d\n", 255);   // 255
printf("%o\n", 255);   // 377
printf("%x\n", 255);   // ff
printf("%#o\n", 255);  // 0377
printf("%#x\n", 255);  // 0xff
printf("%#X\n", 255);  // 0XFF
```

> ⚠️ 注意事项
>
> 1. **int 的字节长度依赖平台**：常见为 4 字节（32 位），但 2 字节（16 位）或 8 字节（64 位）也有可能。不要硬编码 `int` 的字节数，必要时使用 `<stdint.h>` 中的定宽类型（如 `int64_t`）。
>
> 2. **溢出行为**：给 `int` 变量赋一个超出其表示范围的值时，会发生溢出。即使是 64 位编译选项（`-m64`），普通 `int` 仍然可能只有 4 字节，无法存储 64 位整数。需要 64 位整数时应使用 `long long`。
>
> 3. **unsigned 省略 int**：`unsigned` 单独使用时等同于 `unsigned int`，`unsigned short`、`unsigned long`、`unsigned long long` 中的 `int` 同样可以省略。
>
> 4. **char 的特殊性**：`char` 不等同于 `signed char`，其默认符号性由编译器决定。如果代码依赖符号行为，应明确写成 `signed char` 或 `unsigned char`。
>
> 5. **子类型选择建议**：根据实际需要的取值范围选择合适的子类型——`short` 用于 16 位，`long` 用于 32 位，`long long` 用于 64 位。既能节约内存，又能明确表达代码意图。
